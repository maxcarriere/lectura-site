La page **Méthode** explique **comment** les projets Lectura sont produits, en s’appuyant fortement sur l’IA tout en gardant une validation humaine claire.

Cette version propose un résumé opérationnel, aligné sur la vision générale.

## 1. Principe général

La méthode Lectura repose sur une idée simple :

> **Utiliser au maximum les outils existants, en particulier l’IA, pour concentrer le travail humain sur ce qui n’est pas délégable.**

Concrètement :

- l’humain porte la **vision**, fixe les priorités, arbitre et valide ;
- l’IA **exécute, structure, documente et accélère** les différentes tâches.

## 2. Workflow de production

Le cycle typique d’un projet Lectura peut se résumer ainsi :

1. **Idée**  
   Formulation d’une intention : produit, service, lecteur, application, etc.

2. **Cadrage**  
   Rédaction rapide d’un document de référence : objectifs, contraintes, public, périmètre.

3. **Production assistée par IA**  
   Délégation à des agents IA pour :
   - générer du code,
   - structurer des contenus,
   - explorer des options techniques.

4. **Validation humaine**  
   Relecture, ajustements, arbitrages, choix finaux.

5. **Diffusion**  
   Mise en ligne (site, dépôt GitHub, démo…), documentation minimale, possibilité de retours.

## 3. Répartition des rôles

- **Humain**
  - définit la direction ;
  - tranche en cas d’ambiguïté ;
  - vérifie la cohérence globale ;
  - garantit le respect du cadre légal.

- **IA**
  - propose plusieurs options ;
  - exécute les tâches répétitives ou techniques ;
  - aide à documenter ce qui a été fait et pourquoi.

## 4. Documentation

Chaque contribution importante doit produire à la fois :

- un **résultat exploitable** (code, page, structure…) ;
- une **courte explication** des choix, limites et pistes d’amélioration.
